JSON vs XML and JSON Syntax in Hindi

JSON vs XML

Feature / Aspect JSON (JavaScript Object Notation) XML (eXtensible Markup Language
Format type Lightweight, text-based Text-based, markup language
Data Representation Key-Value pairs Tags (Start & End) based
Readability Simple, easy to read & write Verbose, थोड़ा complex पढ़ने में
Data Size Compact, smaller size Larger size due to opening/closing tags
Parsing Fast (built-in support in JS) Slower compared to JSON
Supported Types Strings, Numbers, Boolean, Arrays, Objects Mainly Strings (Data types explicitly नहीं होते, Schema/XSD चाहिए)
Syntax Curly braces { } और Square brackets [ ] use करता है Angle brackets < > use करता है
Comments Allowed (but officially discouraged) Allowed (<!-- -->)
Schema Support JSON Schema (modern, flexible) DTD और XML Schema (XSD) support
Use Cases APIs, Web Apps, Config files, Databases Documents, Configs, Data Exchange (legacy)
Popularity Today Modern web & mobile apps prefer JSON XML अभी भी कुछ enterprise systems और पुराने standards में use होता है

JSON ज्यादा simple, fast और lightweight है, इसलिए modern applications और APIs में ज्यादा use होता है।

XML थोड़ा verbose है लेकिन अभी भी enterprise systems, configuration और कुछ पुराने software standards में use होता है।

JSON Syntax Rules

1. Data Format

  • JSON data हमेशा key-value pairs में होता है।
  • Example: "name": "Rahul"

2. Curly Braces { }

  • Curly braces का use object को define करने के लिए होता है।
  • Example: { "name": "Rahul", "age": 21 }

3. Square Brackets [ ]

  • Square brackets का use array define करने के लिए होता है।
  • Example: [ "HTML", "CSS", "JavaScript" ]

4. Keys (Property Names)

  • Keys हमेशा double quotes (" ") में लिखी जाती हैं।
  • Example: "course": "Computer Science"

5. Values

JSON में values निम्न type की हो सकती हैं:

  • String → "Rahul"
  • Number → 101
  • Boolean → true / false
  • Array → [ ]
  • Object → { }
  • Null → null

6. Commas

Multiple key-value pairs या array items को comma ( , ) से separate किया जाता है।

7. No Trailing Comma

JSON में last item के बाद कभी comma नहीं लगाना चाहिए।
