XML DTD and XML Schema in Hindi

XML DTD

XML DTD का use XML document की structure और rules define करने के लिए किया जाता है। यानी DTD बताता है कि एक XML document में कौन-कौन से elements होंगे, उनका order क्या होगा, कौन से elements mandatory हैं और कौन optional.

DTD XML document को validate करने में मदद करता है। अगर कोई XML file DTD में दिए गए rules को follow करती है तो उसे valid XML कहा जाता है। अगर सिर्फ syntax rules follow किए गए हैं लेकिन DTD follow नहीं हुआ तो वो सिर्फ well-formed XML कहलाएगा।

DTD को XML document के अंदर internal रूप से लिखा जा सकता है या external file के रूप में भी link किया जा सकता है। Internal DTD XML file में सीधे declare किया जाता है जबकि external DTD को अलग file में define करके reference दिया जाता है।

DTD में elements, attributes और entities को declare किया जाता है। इसके जरिए यह control किया जा सकता है कि कौन सा element किस element के अंदर आएगा और किस element में कौन सा attribute होगा।

DTD XML applications और parsers को यह समझने में मदद करता है कि XML document को कैसे read और interpret करना है। खासकर जब अलग-अलग systems में data exchange होता है तो DTD data की consistency और correctness सुनिश्चित करता है।

DTD XML के साथ एक पुराना standard है, लेकिन आजकल XML Schema (XSD) ज्यादा popular हो चुका है क्योंकि वो DTD से ज्यादा powerful और flexible है। फिर भी XML DTD XML की validity समझने के लिए बहुत important concept है।

XML Schema

XML Schema को XML documents की structure और rules define करने के लिए use किया जाता है। यह DTD की तरह काम करता है लेकिन उससे कहीं ज्यादा powerful और flexible होता है। XML Schema को अक्सर XSD कहा जाता है।

XML Schema XML document के लिए यह specify करता है कि कौन-कौन से elements होंगे, उनका data type क्या होगा, कौन से attributes use किए जाएंगे और उनका format कैसा होगा। इससे XML document को ज्यादा strict और detailed तरीके से validate किया जा सकता है।

DTD केवल basic validation देता है, जबकि XML Schema में data types (जैसे string, integer, date, boolean) specify किए जा सकते हैं। इसका मतलब है कि XML Schema यह ensure करता है कि सिर्फ सही प्रकार का data ही XML document में store किया जाए।

XML Schema खुद भी XML format में लिखा जाता है। इस वजह से इसे पढ़ना और maintain करना आसान होता है और XML tools इसे आसानी से process कर सकते हैं।

Schema का सबसे बड़ा फायदा यह है कि यह namespaces support करता है। यानी एक XML document में multiple vocabularies को बिना किसी conflict के manage किया जा सकता है।

XML Schema XML documents को well-structured, consistent और reliable बनाने में मदद करता है। इसी कारण modern applications और web services (जैसे SOAP) में DTD की जगह XML Schema का use ज्यादा किया जाता है।
